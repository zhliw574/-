2.3 前后端接口设计（API）
前后端约定 RESTful 风格 API，数据交换格式为 JSON。

功能	方法	URL	请求体示例	响应示例
用户注册	POST	/api/register	{"username":"zhang","password":"123","email":"a@b.com"}	{"code":200,"message":"注册成功"}
用户登录	POST	/api/login	{"username":"zhang","password":"123"}	{"code":200,"token":"xxx","user_id":1}
添加流水	POST	/api/transactions	{"type":"支出","category":"餐饮","amount":45.5,"date":"2025-04-15"}	{"code":200,"trans_id":101}
获取流水列表	GET	/api/transactions	?user_id=1	{"code":200,"data":[...]}
编辑流水	PUT	/api/transactions/<id>	{"amount":50.0}	{"code":200,"message":"更新成功"}
删除流水	DELETE	/api/transactions/<id>	—	{"code":200,"message":"删除成功"}
月度报表	GET	/api/report/monthly	?year=2025&month=4&user_id=1	{"total_income":8000,"total_expense":3200,"balance":4800}
分类统计	GET	/api/stat/category	?type=支出&year=2025&month=4&user_id=1	[{"name":"餐饮","value":1200},{"name":"交通","value":300}]
设置预算	POST	/api/budget	{"category":"餐饮","month":"2025-04","limit":2000}	{"code":200,"message":"预算设置成功"}

三、数据库设计
3.1 数据库概述
数据库名称：fund_management

字符集：utf8mb4

存储引擎：InnoDB

3.2 表结构设计
（1）用户表（user）
字段名	类型	约束	说明
user_id	INT	PRIMARY KEY AUTO_INCREMENT	用户唯一标识
username	VARCHAR(32)	NOT NULL UNIQUE	用户名
password	VARCHAR(64)	NOT NULL	加密后的密码
email	VARCHAR(64)		邮箱
income_total	DECIMAL(10,2)	DEFAULT 0	累计总收入
budget_food	DECIMAL(10,2)	DEFAULT 0	餐饮类预算
budget_non_food	DECIMAL(10,2)	DEFAULT 0	非餐饮类预算

（2）资金流水表（transaction）
字段名	类型	约束	说明
trans_id	INT	PRIMARY KEY AUTO_INCREMENT	流水唯一标识
user_id	INT	NOT NULL FOREIGN KEY	所属用户
trans_type	ENUM('收入','支出')	NOT NULL	交易类型
amount	DECIMAL(10,2)	NOT NULL	金额
category	VARCHAR(32)	NOT NULL	分类
trans_date	DATE	NOT NULL	交易日期
remark	VARCHAR(255)

（3）管理员表（admin）
字段名	类型	约束	说明
admin_id	INT	PRIMARY KEY AUTO_INCREMENT	管理员唯一标识
admin_name	VARCHAR(32)	NOT NULL UNIQUE	管理员账号
password	VARCHAR(64)	NOT NULL	加密后的密码
3.3 基础 SQL 操作模板
-- 用户插入
INSERT INTO user(username,password,email,income_total,budget_food,budget_non_food) 
VALUES (?,?,?,?,?,?);

-- 用户查询
SELECT * FROM user WHERE username = ?;

-- 流水插入
INSERT INTO transaction(user_id,trans_type,amount,category,trans_date,remark)
VALUES (?,?,?,?,?,?);

-- 流水查询（某用户所有记录）
SELECT * FROM transaction WHERE user_id = ?;

-- 月度支出分类统计
SELECT category, SUM(amount) as total 
FROM transaction 
WHERE user_id = ? AND trans_type = '支出' AND trans_date BETWEEN ? AND ?
GROUP BY category;

-- 管理员登录验证
SELECT * FROM admin WHERE admin_name = ?;
